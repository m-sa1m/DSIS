Trigger: auto_generate_alert()

When a detected_object is inserted with threat_level = 'High', the trigger automatically creates an alert with severity = 'Critical' and status = 'New'.

Stored Procedures

Security Measures

  • Passwords hashed with bcrypt (10 salt rounds)
  • JWT tokens with 7-day expiry
  • RBAC middleware checks role on every request
  • Helmet.js for HTTP security headers
  • Zod validation on all request bodies
  • Audit trail for all admin actions

Implementation

Backend: auth.js middleware verifies JWT → rbac.js middleware checks role_name against allowed roles array → Controller executes or returns 403.

Normalization — 1NF to BCNF

1NF

  • All attributes are atomic (no arrays/nested data)
  • Each table has a primary key
  • No repeating groups

2NF

  • All non-key attributes fully depend on the entire primary key
  • No partial dependencies (all PKs are single-column SERIAL)

3NF

  • No transitive dependencies
  • Example: user role info stored in separate roles table, not in users

BCNF

  • Every determinant is a candidate key
  • roles: role_id → role_name (PK) and role_name → role_id (UNIQUE)
  • users: user_id → all attrs, email → all attrs (UNIQUE)
  • Documented in queries.sql (Q14)
